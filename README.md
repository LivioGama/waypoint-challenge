# waypoint-iep-mcp

MCP server that turns one student's IEP + tomorrow's lesson into a classroom-ready packet of modifications.

Special-ed teachers spend ~7 hours a week adapting materials for students on IEPs. This compresses one student × one lesson down to 30 seconds — without watering down the standard, and with every modification carrying a citation back to a specific IEP element and lesson part (the audit trail IDEA requires).

Built for the **Waypoint Learning Special-Education Challenge**.

- **Live demo:** https://waypoint-challenge.vercel.app
- **Remote MCP server:** `https://waypoint-challenge.vercel.app/api/mcp`
- **Teacher packets:** [Jasmine — Health Impairment](examples/jasmine_community-essay_PACKET.md) · [Marcus — Dyslexia](examples/marcus_community-essay_PACKET.md) · [Elena — ASD](examples/elena_community-essay_PACKET.md) · [Devon — ED/anxiety](examples/devon_community-essay_PACKET.md)
- **Spanish parent letters:** [Jasmine](examples/jasmine_community-essay_PARENT_es.md) · [Marcus](examples/marcus_community-essay_PARENT_en.md) (en) · [Elena](examples/elena_community-essay_PARENT_es.md) · [Devon](examples/devon_community-essay_PARENT_es.md)
- **PDF intake:** [data/students/jasmine_from_pdf.json](data/students/jasmine_from_pdf.json) — generated by `bun run parse-iep ./iep` against the original challenge PDF, no manual edits.
- **Audit eval:** [examples/EVAL.md](examples/EVAL.md) — all 8 packets scored 1–10 on 5 dimensions by an independent judge. Avg **42.1/50**, range 37–45.
- **vs naive baseline:** [examples/BASELINE_COMPARISON.md](examples/BASELINE_COMPARISON.md) — single-shot GPT vs Waypoint loop on the same input. **Citation match rate: 36% → 89.5% (+147%).** Defensibility, not just polish.
- **Tool-augmented vs full-context A/B:** [examples/AB_COMPARISON.md](examples/AB_COMPARISON.md) — same generation pipeline, only `USE_TOOL_AUGMENTED` flipped. Tool-augmented wins on grounding (9 vs 8) and student-fit (9 vs 7).

## What it produces

A 3-part packet for one student × one lesson:

1. **Cheat sheet** — pre-bell prep checklist, early-warning signals, recovery script, praise stems.
2. **Modified lesson plan** — every block of the original lesson with the modified step, the IEP element it grounds in (quoted), and the materials swap.
3. **Modified student materials** — anchor card, modified MCQs (same questions, same standards, same answers, with a "what this is really asking" line), sentence-stem scaffold, and an index table mapping every modification to (IEP element, lesson part).

Every action is wrapped in `[[NEW]]`, `[[CHANGED]]`, or `[[SAME]]`. The web UI color-codes these — green / yellow / gray.

## Architecture

**Why MCP.** The teacher's workflow lives in Claude Desktop / Cursor. The IEP and lesson are reference material the model reaches into, not carries in every prompt.
- **Resources:** one per student IEP, one per lesson, structured + full text.
- **Tools:** granular access (`get_iep_section`, `get_lesson`, ...) and heavy generation (`generate_modifications`).

**Data shape.** Each IEP is split on the seams an IEP team thinks in: present level by subject, accommodations by IDEA category, goals with their next benchmark scaffolded out. Adding a student is one JSON file. The repo ships with **4 students across distinct disability categories** (Health Impairment, Dyslexia, ASD, ED).

**Refinement loop.** `generate_modifications` runs **4 parallel candidate drafts + 1 judge = 5 model calls**. Each candidate is told to bring a distinct angle. The judge applies a 5-criteria rubric (grounding / actionability / fidelity / student-fit / prose) and returns the winner with a one-sentence reason.

**Tool-augmented generation.** Each candidate runs in a tool-use loop: instead of dumping the full IEP + lesson into context, the model gets a small context block (student name, lesson title, standards, available section names) and calls `get_iep_section` and `get_lesson_excerpt` mid-draft to fetch exactly the goals / accommodations / paragraphs it needs. Audit-able: the trace shows *which* benchmarks the model actually consulted. Toggle via `USE_TOOL_AUGMENTED` in `src/refine.ts`.

**Citation verifier in the loop.** Every candidate is verified BEFORE the judge sees it. The verifier extracts every quoted phrase + structural reference (`Paragraph N`, `MCQ #N`), greps against the source IEP JSON and lesson text, and computes a match rate per candidate. Candidates below `MIN_CITATION_RATE = 0.4` are auto-rejected from the judge pool. Surviving candidates' rates are passed to the judge prompt so it can favor verifiable ones. The system self-corrects hallucinations instead of waiting for a human to catch them. Also exposed standalone via `bun run verify <packet>` and as the `verify_citations` MCP tool.

**Parent-facing translation.** `generate_parent_summary({student_id, lesson_id, language: "es"})` distills the teacher packet into a one-page parent letter in Spanish (or any language). Plain language, one concrete home-practice suggestion, signed by the teacher.

**PDF intake.** `bun run parse-iep <pdf>` runs `pdf-parse` over a real IEP PDF and asks the model to coerce extracted text into the canonical JSON schema. Tested against the original challenge starter PDF — output committed at `data/students/jasmine_from_pdf.json`.

**Two transports, one factory.** `buildServer()` powers both `StdioServerTransport` (local Claude Desktop) and `StreamableHTTPServerTransport` (remote MCP via Vercel). Same code, two transports.

**The "layman prompt".** The system prompt forces plain language, hard rules (no generic strategies, no watering down the standard), and two-sided citation discipline (every modification names IEP element + lesson part). Modifications that can't carry the citation get cut by the judge.

## MCP API

### Resources

| URI pattern | Returns |
|---|---|
| `iep://<student_id>/structured` | Full structured IEP |
| `lesson://<lesson_id>/full-text` | Raw lesson text |
| `lesson://<lesson_id>/meta` | Lesson metadata |

### Tools

| Name | Args | Returns |
|---|---|---|
| `list_students` | — | All students |
| `list_lessons` | — | All lessons |
| `list_iep_sections` | — | Section names fetchable via `get_iep_section` |
| `get_iep_section` | `section`, optional `student_id` | One IEP section. Accepts `student`, `disability`, `present_levels[.ela|.math|.behavioral]`, `accommodations`, `annual_goals`, `goal.<n>`, etc. |
| `get_lesson` | optional `lesson_id` | Full lesson text |
| `generate_modifications` | optional `student_id`, `lesson_id`, `return_candidates` | Runs the 4-draft + judge loop. Returns the winning packet. |
| `verify_citations` | `packet`, optional `student_id`, `lesson_id` | Greps every quoted phrase in a packet against the source IEP JSON + lesson text. Validates `Paragraph N` / `MCQ #N` references. Returns a structured match report. |
| `parse_iep_pdf` | `pdf_base64` | Extracts text from a base64-encoded IEP PDF and coerces it into the canonical structured-IEP JSON shape. Closes the loop on real-world PDF intake. |
| `generate_parent_summary` | `student_id`, `lesson_id`, optional `language` (default `"es"`), optional `packet` | Produces a one-page parent-facing letter explaining what is happening tomorrow and one home-practice suggestion. Spanish by default. |

## Run locally

```bash
git clone https://github.com/LivioGama/waypoint-challenge.git
cd waypoint-challenge
bun install
cp .env.example .env  # add ANTHROPIC_API_KEY or OPENAI_API_KEY
```

```bash
bun test                  # 18 tests
bun run web               # http://localhost:8787 — full web UI
bun run demo              # CLI: writes packet to examples/
bun run eval              # post-hoc judge scores all packets in examples/, writes EVAL.md
bun run verify <packet>   # citation-verifier: every quote checked against IEP + lesson
bun run parse-iep <pdf>   # PDF → structured IEP JSON
bun run dev               # raw stdio MCP server
bun run inspect           # MCP Inspector
```

## Hook into Claude Desktop

Edit `~/Library/Application Support/Claude/claude_desktop_config.json`.

**Local stdio:**
```json
{
  "mcpServers": {
    "waypoint-iep": {
      "command": "bun",
      "args": ["run", "/absolute/path/to/waypoint-challenge/src/server.ts"],
      "env": { "ANTHROPIC_API_KEY": "sk-ant-..." }
    }
  }
}
```

**Remote HTTPS** (no install):
```json
{
  "mcpServers": {
    "waypoint-iep-remote": {
      "url": "https://waypoint-challenge.vercel.app/api/mcp"
    }
  }
}
```

Then ask: *"Use the waypoint-iep server to generate Marcus's packet for the community lesson."*
